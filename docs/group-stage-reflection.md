# 小组赛预测复盘反思

Last updated: 2026-07-01

## 当前统计

- 小组赛胜平负方向：47/72 = 65.3%。
- 小组赛前三比分命中：22/72 = 30.6%。
- 计分规则：胜平负按 90 分钟结果；比分命中只要前三预测比分包含实际比分就算命中。

## 主要错误原因

1. 平局保护不够。
   小组赛里有不少强弱差不大、双方都能接受低风险结果的比赛，模型仍然偏向实力更强一方，导致平局被低估。

2. 最后一轮出线动机考虑太晚。
   已经基本出线的球队会降节奏、轮换或保守；必须抢分/净胜球的球队会更激进。之前模型更多看实力、Elo、近期状态，没有把“积分形势”提前放到比分分布里。

3. 热门零封和大胜比分偏高。
   早期模型容易给出 2-0、3-0、4-0，但实际很多弱队仍能进一球，或者强队领先后控节奏，比分落到 2-1、1-1、1-0。

4. 东道主和比赛环境没有单独建模。
   墨西哥这类主场氛围强、节奏变化大的比赛，不能只按中性场实力处理；天气延迟、红牌、主场压力都会改变比赛路径。

5. 缺少首发、轮换、伤停和临场市场确认。
   小组赛最后阶段尤其明显：轮换和保守策略会直接影响进球数和双方进球概率。

## 后续规则

- 小组赛最后一轮必须先算积分形势，再算 xG 和比分。
- 已出线或打平即可接受的球队：降低节奏，提高 0-0、1-1、1-0、0-1 权重。
- 必须赢球或抢净胜球的球队：提高进攻 xG，同时提高被反击/双方进球风险。
- 热门胜率没有达到强信号时，不再把 2-0/3-0 当默认主比分。
- 比分预测继续看前三比分命中率，不只看单个最可能比分。
- 赛果刷新必须优先自动化：openfootball 慢时，使用已配置的网络赛事页自动补抓；只有网络源缺失或无法稳定解析时才需要人工补源。
